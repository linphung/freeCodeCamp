var vs let

var declares variables globally and are hoisted eg. undefined error if variable is used before declaration

let declares variables that are limited to the block statement and are not hoisted eg. ReferenceError if variable is used before declaration

note: variables declared by let cannot be redeclared

And most importantly, const are constant variables that are set to not change EVER for good reason (whatever the programmer decides that reason is)

functions

functions can be declared by the simple skeleton

function foo() { }

parameters need not to be declared first before using, for example

function foo(param) { }

multiple parameters can even be assigned as such

function foo(param1, param2) { }

Printing

Depending on what language you're used to, you might expect to use the term "print" for printing out results. This is not the case in JavaScript. We use console.log() to do so.

function foo(param) { console.log(param); }

Adding

Thankfully, JavaScript seems to have a grasp on what programmers want to do when we add things. We can add different types of numbers and strings together no problem.

function foo() { var a = 5; var b = 3; var c = a + b; console.log(c); var line1 = 'Hello '; var line2 = 'World!'; console.log(line1 + line2); }

Incrementation is also very similar to other languages

function foo(num){ num++; // num += 1 num--; // num -= 1 }
